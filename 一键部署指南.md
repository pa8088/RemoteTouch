## 📋 准备工作（5分钟）

### 1. 注册 GitHub 账号（如果没有）
```
访问: https://github.com/signup
免费注册，填写邮箱和密码即可
```

### 2. 创建访问令牌（GitHub Token）

**为什么用 Token 而不是密码？**
- ✅ 更安全（可以随时撤销）
- ✅ 可以限制权限（只给必要的权限）
- ✅ 不会暴露真实密码

**创建步骤**：

1. 登录 GitHub 后访问：https://github.com/settings/tokens/new

2. 填写信息：
   ```
   Note: RemoteTouch Deploy
   Expiration: 90 days（90天后过期）

   Select scopes（权限选择）:
   ✅ repo（勾选这个及其所有子选项）
      ✅ repo:status
      ✅ repo_deployment
      ✅ public_repo
      ✅ repo:invite
      ✅ security_events
   ```

3. 滚动到底部，点击 `Generate token`

4. **⚠️ 重要**：复制生成的 Token（形如 `ghp_xxxxxxxxxxxx`）
   - **只显示一次，请立即保存到记事本！**
   - 如果丢失，需要重新生成

## ❓ 常见问题

### Q1: 提示"git 命令不存在"

**解决方法**：
- **Windows**: 下载安装 https://git-scm.com/download/win
- **macOS**: 运行 `brew install git`（需先安装 Homebrew）
- **Linux**: 运行 `sudo apt install git`（Ubuntu/Debian）

### Q2: 提示"Token 无效"或"权限不足"

**解决方法**：
1. 检查 Token 是否完整复制（包括 `ghp_` 前缀）
2. 确认创建 Token 时勾选了 `repo` 权限
3. Token 可能已过期，重新生成一个

### Q3: 提示"仓库已存在"

**解决方法**：
- 使用不同的仓库名称，或
- 删除 GitHub 上的同名仓库后重试

### Q4: 推送失败

**解决方法**：
```bash
# 清理本地 git 配置重试
rm -rf .git
bash auto-deploy.sh
```

### Q5: Token 泄露了怎么办？

**解决方法**：
1. 访问 https://github.com/settings/tokens
2. 找到对应的 Token
3. 点击 `Delete` 删除
4. 重新生成新的 Token

## 🔒 安全提示

✅ **安全的做法**：
- 使用 GitHub Token（可以随时撤销）
- Token 只给必要的权限（repo）
- Token 设置过期时间（90天）
- Token 妥善保存，不要分享给他人

❌ **不安全的做法**：
- 使用 GitHub 密码
- Token 给过多权限
- Token 永不过期
- Token 发到公开的地方（论坛、聊天记录）
